# Pilares: Decomposição 

 

Primeiro passo da resolução de problemas dentro do conceito de pensamento computacional. 

*"Dado um problema complexo, devemos quebra-lo em problemas menores. Portanto, problemas mais fáceis e gerenciáveis."*



- **ESTRATÉGIAS** - processo de quebrar e determinar parte menores e gerenciáveis - ANÁLISE. 

  **Você precisa:** estudar, explorar, realizar um exame bem detalhado, decompor em elementos constituintes - quebrar em problemas menores e então exame os problemas menores. 

- **SÍNTESE**: reunir elementos distintos de um único grande elemento, dentro de um processo de reconstrução, fundindo os elementos de maneira coerente e dando um sentido na solução. 

**ORDEM DE EXUÇÃO DE TAREFAS MENORES:** sequencial ou paralelo. 

- **Sequencial**: dependência entre tarefas, executadas em fila. Para resolver um problema é necessário resolver outro problema.
- **Paralelo**: ganha em eficiência e tempo - tarefas podem ser executadas concomitantemente. Tarefas são resolvidas sem alguma ordem sequencial e depois são unidas. 

**NÃO BASTA APLICAR** - tem que entender;

# Pilares: Padrões 



**Reconhecimento de Padrões.** 

- Modelo base
- Estrutura invariante
- Repetição 



**Ex:** modelo de cadeira, tem encosto, pés, assento. Tudo vai variar e temos um exigência mínima para ser reconhecida como cadeira. 

**Como reconhecer padrões?** através da detecção de **similaridades** e **diferenças** entre contextos e objetos. 

**Ex:** redes sociais. 

- Imagem grande > precisa ser comprimida > salvas no servidor de dados -> Processo utilizado por plataformas diferentes. 



**Por que determinar padrões?** 

- Temos uma vantagem fazendo isso você consegue generalizar, com o objetivo de obter resolução para problemas diferentes. 

**Como um ser humano faz isso?** 

- Grau de similaridade, grupos conhecidos x objetos, desconhecidos.
- Ser humano é subjetivo.

**Como um computador reconhece os padrões?**

- Representação de atributos

- Aprendizado - conceito associado ao objeto. 

- Armazenar dados para consultas posteriores a fim de determinar objetos

- Regras de decisão.

  

**RESUMO** - detecção de padrão vem da ideia de você extrair características a fim de classificar os seus dados. Conseguindo atuar com diferentes métodos e aplicar em diferentes contextos. 



**APLICAÇÕES QUE UTILIZAM O RECONHECIMENTO DE PADRÕES**

- Classificação de dados
- Reconhecimento de imagem: placa de carro.
- Reconhecimento de fala: idioma que esta sendo falado.
- Análise de cenas
- Classificação de documentos. 



**ÁREAS QUE UTILIZAM O RECONHECIMENTO DE PADRÃO**

- Machine Learning.
- Redes Neurais.
- Inteligência Artificial.
- Ciência de Dados. 



# Pilares: Abstração



**Entendendo os conceitos.** 

- **Abstrair**: observar, um ou mais elementos, avaliando características e propriedades em separado. 
- **Abstração**: processo intelectual de isolamento de um objeto da realidade. (mundo abstrato)
- **Generalizar**: tornar-se geral, mais amplo, extensão. 



**Generalização X Abstração** 

Generalização, na lógica, é a operação intelectual que consiste em reunir numa classe geral, um conjunto de seres fenômenos similares. 

**Como classificar os dados?** 

- Características
- Pontos essenciais
- Generalizar x Detalhar.

**EX:** representação de dados. 

**Características**: nome, matricula, endereço, campus, etc. 

Alguns pontos são essenciais: identificação dos alunos, nome, matricula, endereço, campus, etc.

Outros detalhes sao deixados de lado.

**Abstração**: ideia de abstrair um conceito e generalizar o problema - torna-lo reutilizado por cenários distintos mas com similaridade entre si. 

Exemplo do terreno, qual o menor caminho para começar a limpar. 



# Pilares: Algoritmo



**CONCEITO**:

-  **COMPUTAÇÃO:** conjunto das regras e procedimentos lógicos perfeitamente definidos que levam à solução de um problema em um número finito de etapas.
- **MATEMÁTICA:** sequência finita de regras, raciocínios ou operações que, aplicada a um número finito de dados, permite solucionar classes semelhantes de problemas.

**Computador:** 

- Energia 
- Trabalhador
- Eficiência 
- Rapidez

Porém, ele não opera sozinho! Precisa de instruções detalhadas. 

O computador recebe, manipula e armazena dados. 

Programas - sao construidos pelas instruções. 

Processo de resolução de programas "Step by setp"

- Desenvolvimento do programa: análise, estudo e definição dos dados de entrada e saída. 

  Descreve o problema por meio de ferramentas narrativas, fluxograma. 

  Codificação.

**Exemplos do que é um algoritmo.** 

São sequencias de instruções que você realizar para uma determinada tarefa. 

- Preparar um sanduiche

- Trocar uma lâmpada

- Trajeto ao trabalho

- Fazer uma receita de bolo 

  

**COMO CONSTRUIR UM ALGORITMO?** 

- Compreensão do problema

- Definição de dados de entrada

- Definir processamento

- Definir dados de saída

- Utilizar um método de construção

- Teste e diagnóstico. 

  

**CONSTRUÇÃO DE UM ALGORITMO**

**Narrativa**: utilização da linguagem natural, diversas interpretações possíveis, linguagem natural (nativa)

**Fluxograma**: utilização de símbolos pré-definidos que definem qual o tipo de ação que está sendo realizada. 

**Pseudocódigo**: regras bem definidas, passo a passo a ser executado, Portugol.

# ESTUDO DE CASO 1: perdido. 

 

Uma pessoa perdida em uma floresta, como resolver o problema usando o pensamento computacional? 

1) Identificar mecanismos
2) Recursos comuns
3) Detalhes mais importantes



- **Sobrevivência**: água, comida e abrigo. 

  Problema sobrevivência esta sendo decomposto em três partes.

  Podemos decompor ainda mais os nossos problemas, por exemplo a água. 

   

  - ÁGUA: chuva ou nascente.                  -> ***fogo***
  - COMIDA: frutos, caçar, coletar.          -> ***fogo*** 
  - ABRIGO: proteção, quente e seco.  ->  mapa    -> ***fogo***



Se repararmos, tem um padrão de repetição nessa segmentação, que é o FOGO. 

Fogo é um ponto em comum para todos os segmentos do problema. 

Fogo é o principal recurso a ser utilizado.

Mapa, podemos criar por abstração - observando o meio, florestra, identificando detalhes cruciais relacionados a situação. 

***Desse modo, já utilizamos a decomposição, reconhecimento de padrões, abstração.***
